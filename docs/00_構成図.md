# 0. システム構成図（物理 / 論理）

shake 基盤の全体像を「物理構成」と「論理構成」の2つの視点で示す。

- **物理構成図**: どの機材が・どの拠点に・どう接続されているか
- **論理構成図**: リクエストがどう流れ、各ポート/コンテナがどう連携するか

図は Git 管理しやすい Mermaid で記述している。本ファイルは GitHub 上でそのまま描画される。
見栄えを優先した [D2](https://d2lang.com/) 版のソースも [diagrams/](diagrams/) に併置している（描画方法は末尾参照）。

## 0.1 物理構成図（ノード・拠点・ネットワーク）

```mermaid
flowchart LR
    subgraph Home["自宅ネットワーク (DS-Lite / 外部公開不可)"]
        UPS1["UPS<br/>CyberPower CP1200PFCLCDJP"]
        UPS2["UPS<br/>Geekworm X1202"]
        Router["ホームルーター<br/>Aterm-WG1200HP4"]
        Switch["スイッチングハブ<br/>TP-Link TL-SG605"]
        Pi5["<b>shakeserver / home_node1</b><br/>Raspberry Pi 5 (8GB)<br/>メインノード（アプリ本体）"]
        Pi4["<b>tarakoserver / home_node2</b><br/>Raspberry Pi 4 (4GB)<br/>監視ノード + CI Runner"]
    end

    subgraph Univ["大学ネットワーク (グローバルIP公開点)"]
        PiZero["<b>negitoroserver / proxy_node</b><br/>Raspberry Pi Zero 2 W<br/>プロキシノード"]
    end

    TS(("Tailscale VPN<br/>100.64.0.0/10"))

    subgraph Net["外部インターネット"]
        User["外部ユーザ"]
        Workers["CF Workers<br/>メンテナンス / Failover"]
        R2["Cloudflare R2<br/>(外部バックアップ)"]
    end

    %% データリンク (L2)
    Router --> Switch
    Switch --> Pi5
    Switch --> Pi4

    %% 電源系統
    UPS1 -.->|給電| Router
    UPS1 -.->|給電| Switch
    UPS1 -.->|給電| Pi4
    UPS1 -.->|給電 商用電源側| Pi5
    UPS2 -.->|給電 基板側/二重化| Pi5

    %% Tailscale メッシュ（オーバーレイ）
    PiZero <-.->|Tailscale| TS
    Pi5 <-.->|Tailscale| TS
    Pi4 <-.->|Tailscale| TS

    %% 外部トラフィック / フェイルオーバー / バックアップ
    User -->|HTTP/HTTPS| PiZero
    User -.->|障害時 HTTP 503| Workers
    Pi4 -.->|自動切替 ダウン検知| Workers
    Pi5 -->|日次バックアップ送信| R2

    classDef pi fill:#c8e6c9,stroke:#2e7d32,color:#000
    classDef ups fill:#f3e5f5,stroke:#6a1b9a,color:#000
    classDef cloud fill:#fff3e0,stroke:#ef6c00,color:#000
    class Pi5,Pi4,PiZero pi
    class UPS1,UPS2 ups
    class Workers,R2 cloud
```

自宅にグローバル IP を晒さず、大学拠点の Pi Zero 2 W を公開ゲートウェイにし、3 台を
Tailscale メッシュ VPN（ポート開放不要・WireGuard ベース）で接続している。
詳細は [02_ネットワーク基盤.md](02_ネットワーク基盤.md) を参照。

443 番一本に HTTPS と Minecraft を多重化（sslh → Nginx stream → Proxy Protocol で実 IP 復元）。
内部サービス（5432/3000/9090 等）はすべて Tailscale + 緊急 LAN からのみ到達可能で、
外部公開は proxy_node の 80/443 だけ。UFW の IP 戦略は [02_ネットワーク基盤.md](02_ネットワーク基盤.md)、
監視データフローは [04_監視・ログ・通知基盤.md](04_監視・ログ・通知基盤.md) を参照。

### 再生成方法
ソースは [diagrams/physical.d2](diagrams/physical.d2) / [diagrams/logical.d2](diagrams/logical.d2)。

```bash
# インストール (一度だけ)
curl -fsSL https://d2lang.com/install.sh | sh -s --

# 描画 (SVG 出力)
d2 docs/diagrams/physical.d2 docs/diagrams/physical.svg
d2 docs/diagrams/logical.d2  docs/diagrams/logical.svg

# 編集しながらブラウザでライブプレビュー
d2 --watch docs/diagrams/physical.d2
```

> PNG で書き出す場合（`d2 physical.d2 physical.png`）は headless Chromium が必要。
> SVG はベクタ形式で GitHub 上でもそのまま表示されるため、通常は SVG 出力で十分。
