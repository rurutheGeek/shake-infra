# ディレクトリ構造解説リファレンス

## 本書の目的
各章末の「関連ファイル一覧」と README のツリーに分散しているディレクトリ・ファイルの役割を、検索可能な単一文書として集約したリファレンスである。「このファイルは何のためにあるのか」「この設定はどこを編集するのか」を一覧から引けることを目的とする。

各エントリには、より詳しい解説がある章へのリンクを付す。

---

## 1. リポジトリ全体像

```text
infra/
```

### 3.3 local_config/ — 個人設定・機密（平文は Git 管理外）

```text
local_config/
├── ansible/
│   ├── inventory.ini          # 管理対象ノードの定義（実体）
│   ├── vars.yml               # 公開変数（IP/ドメイン/バケット名）
│   └── credentials/
│       ├── vault.yml          # 暗号化機密（AES256）
│       ├── .vault_pass        # Vault 復号パスフレーズ
│       ├── id_rsa             # SSH 秘密鍵
│       └── key.pem            # SSL 秘密鍵（暗号化）
└── terraform/
    ├── terraform.tfvars       # Terraform 用機密（PAT/トークン/R2 鍵）
    └── backend.hcl            # R2 Remote State バックエンド認証
```

| local_config/ansible/vars.yml | ドメイン名、R2 バケット名、Account ID 等の公開値 | [01章 §1.9](01_IaC環境構築と初期プロビジョニング.md) |

## 5. output/ — 運用ログ・作業日誌

`output/` は IaC の適用対象ではなく、運用作業の記録（日付ファイル名の作業日誌、`infrastructure_overview.md` などの俯瞰メモ、評価記録）を蓄積する領域である。構成管理コードからは参照されない。

この領域には作業の過程で生成された一時ファイルが混在することがあるため、機密を含むファイル（vault.yml 等）が誤って置かれていないかを定期的に点検し、不要なものは整理する。
